/api/user/register

회원가입을 진행합니다.

- 요청 예시:

        {
        "username":"캡스톤싫어",
        "email":"example@.com",
        "password":"someCamelCasePasswordExample"
        }

- 응답 예시:

        {
        "Success": "캡스톤싫어 님, 가입을 환영합니다!"
        }

/api/user/login

로그인을 진행합니다.

- 요청 예시:

        {
        "email":"example@.com",
        "password":"someCamelCasePasswordExample"
        }

- 응답 예시:

        {
        "access_token":
        "액세스 토큰 값",
        "refresh_token":
        "리프레시 토큰 값"
        }

/api/user/refresh

리프레시 토큰을 받아 새로운 액세스 토큰을 발급합니다.

- 요청 예시:

        헤더에 리프레시 토큰 값을 담아 보냅니다.

- 응답 예시:

        {
        "access_token":
        "액세스 토큰 값",
        "refresh_token":
        "리프레시 토큰 값"
        }

/api/user/{int:유저의 id}/messages

POST : 새로운 쪽지를 작성합니다.

- 요청 예시:

        {
        "message":"안녕?",
        "amount" : 1230,
        "is_moneybag" : true,
        "nickname" : "헤헤"
        }

- 응답 예시:

        {
        "message": "안녕?",
        "amount": 1230,
        "is_moneybag": true,
        "id": 6
        }

GET : 특정 유저의 쪽지를 6개 페이지네이션 한 형태로 조회합니다.

- 요청 예시

?page={숫자} 를 선택해서 페이지를 조회할 수 있습니다. 기본값은 1입니다. 
        http://127.0.0.1:5000/api/user/2/messages?page=2

/api/user/{int:유저의 id}/messages/{int:메시지의 id}

쪽지를 상세 조회합니다.

- 응답 예시:

        {
        "id": 1,
        "amount": 1230,
        "message": "안녕?",
        "is_moneybag": true
        }
